**1. Python 仿真阶段**

这一阶段主要是先验证原理，而不是处理真实采集数据。

* zz_ble_cs_py

  整个早期 Python 仿真工程。
* zz_ble_cs_py/zz_ble_cs_py/src/signal_models.py

  用来构造 1to1 和多目标条件下的理想/半理想信号模型。
* zz_ble_cs_py/zz_ble_cs_py/src/estimators.py

  实现不同的距离估计方法。
* zz_ble_cs_py/zz_ble_cs_py/src/target_matching.py

  多目标情况下的无先验扫描匹配核心逻辑，尤其是 v4 之前的目标匹配方法。
* zz_ble_cs_py/zz_ble_cs_py/src/experiments

  按版本组织的仿真实验脚本，用来逐步验证：

  * 1to1 单目标测距
  * random phase 抵消
  * 多目标叠加下的目标距离判别

这一阶段的作用可以概括为：

先在可控模型里验证“pair phase 抵消随机初相”和“多频点 target matching”是否理论可行。

**2. GNU Radio 平台设计阶段**

这一阶段是把前面的理论和仿真思路，落到可运行的 SDR 平台上。

* 1to1.grc

  1to1 平台 flowgraph。
* 1to1

  1to1 相关生成代码和运行逻辑。
* 1to2.grc

  1to2 平台 flowgraph。
* 1to2

  1to2 相关生成代码和运行逻辑。

这一阶段的作用是：

* 实现 initiator / reflector 的 tone 发送与接收
* 设计多频点扫频流程
* 在 SDR 平台上采集连续 IQ 数据
* 为后续离线分析提供真实实验数据

也就是把“仿真里的信号模型”变成“真实无线链路里的连续采样数据”。

**3. 真实数据采集与离线分析阶段**

这一阶段开始处理真实采集到的数据，并形成最终实验结果。

**3.1 原始数据**

* DATA/DATA

  最原始的连续采集 IQ 数据。

**3.2 连续数据切分成 burst**

* split_continuous_bursts.py

  从连续 IQ 中识别 burst 边界、切出每个频点对应的 burst，并提取每个 burst 的复均值。
* DATA/DATA_burst

  burst 级中间结果。

这一步的作用是：

把连续样本点变成“每个频点一个 burst 观测值”的可分析数据。

**3.3 1to1 离线分析与基线建立**

* analyze_continuous_capture.py

  1to1 主分析脚本，是整个真实数据分析逻辑的核心参考。
* analyze_continuous_capture_simple_match.py

  更简化的 1to1 **match** 分析版本。
* build_data_minus.py

  做 **measurement * conj(calibration)**，得到去除校对组影响后的 1to1 数据。
* DATA/DATA_minus

  去除 calibration 后的 1to1 数据与测距结果。

这一步的作用是：

* 用真实数据验证 1to1 测距是否成立
* 建立每个距离组自己的 1to1 基线距离
* 同时确定真实系统里需要的关键预处理：
  * 异常频点剔除
  * 中心频率附近坏频点处理
  * 段间 offset 拼接
  * 最终 **match** 测距

**3.4 1to1 random phase 验证**

* build_data_random.py

  在 1to1 基线数据上加入同频点相反随机相位差，验证 pair 后是否抵消。
* DATA/DATA_random

  加入 random phase 后的 1to1 数据。

这一步的作用是：

在真实数据基础上确认“random phase 虽然被注入，但 pair 后仍能抵消”，说明 1to1 原理在离线链路中成立。

**3.5 1to2 数据合成与测距**

* build_data_1to2.py

  从两个 1to1 数据组合成 1to2 数据，并对两个 reflector 分别测距。
* DATA/DATA_1to2_template

  当前干净的、用于论文的 1to2 全量实验结果目录。

这一阶段的最终做法是：

* 先把两个距离组的 initiator-from-reflector 做复数叠加，构造 1to2 mixed initiator
* 再分别与目标 reflector-from-initiator 配对，形成 target-specific pair
* 再使用该目标自己在 1to1 中学到的处理模板：
  * 保留频点集合
  * 段间 offset 规则
* 最后进行 1to2 测距

**3.6 误差统计与实验结果汇总**

* plot_1to2_error_distributions.py

  根据 1to2 总汇总生成误差表和图。
* DATA/DATA_1to2_template/summary_all.json

  1to2 全量结果总汇总。
* DATA/DATA_1to2_template/error_plots

  误差统计图与 CSV 表。
* DATA/DATA_1to2_template/error_plots/method_summary.csv

  各种 1to2 测距方法总体效果对比。

这一步的作用是：

对不同 1to2 测距方法进行统一评估，并得到最终论文可用的误差统计。

**一句话概括这条时间线**

* 第一阶段：用 zz_ble_cs_py 做理论与算法仿真
* 第二阶段：用 1to1.grc 和 1to2.grc 搭建 GNU Radio 平台并采集真实数据
* 第三阶段：从 DATA/DATA 出发，经 burst 分割、1to1 基线建立、random phase 验证、1to2 合成与误差分析，得到最终实验结果 DATA_1to2_template
